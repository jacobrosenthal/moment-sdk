Notification

Moment. Notification

new Notification(uid, category, event)

Source:
  • notifications.js, line 25

Represents a notification from a connected and paired iPhone or Android device. Due to the way the Apple Notification Center Service is structured, the notification event that is fired when a new notification appears on the connected phone does not provide all of the attributes for the notification. Instead, you will receive a category (e.g. Incoming Call), an event (e.g. Added/Modified/Removed), and a unique identifier for the notification that allows you to request the additional attributes if needed. The additional attributes include the Title, Subtitle, and App Identifier that produced the notification.

Typically, you will not use the constructor for this class. The class will be instantiated internally, and you will interact with Notification objects in response to the relevant events that are triggered.

Parameters:
Name Type Description
uid Number

A unique integer identifier for the notification
category String

Notification category (e.g. Missed Call)
event String

The event (added, modified, or removed)
